# Evaluación Final – Módulo 2 (APIs y SQL)

## Objetivo del repositorio

Este repositorio contiene la **resolución de la Evaluación Final del Módulo 2**, cuyo objetivo es **evaluar mis conocimientos y habilidades prácticas en**:

* Extracción de datos desde una **API REST**
* Manipulación y estructuración de datos con **Python**
* Diseño y creación de **bases de datos relacionales en MySQL**
* Inserción de datos respetando **integridad referencial y claves foráneas**
* Resolución de consultas SQL de análisis y explotación de datos
* Uso de una base de datos relacional (**Sakila**)

La evaluación se divide en **dos ejercicios**, cada uno con archivos y objetivos claramente diferenciados.

## Ejercicio 1 – Base de datos de películas desde una API

### Objetivo

Extraer información de películas desde una API, procesarla con Python y almacenarla en una base de datos MySQL con un diseño relacional correcto. Posteriormente, obtener información relevante mediante consultas SQL.

---

### Archivo: `bda-modulo-2-evaluacion-ejercicio1-extraccion-datos-peliculas.ipynb`

**Tipo:** Jupyter Notebook (Python)

**Cómo usarlo:**

1. Abrir el archivo con Jupyter Notebook o VS Code
2. Ejecutar las celdas en orden
3. Asegurarse de tener instaladas las librerías necesarias (`requests`, `pandas`)

---

### Archivo: `bda-modulo-2-evaluacion-ejercicio1-bbdd-peliculas.sql`

**Tipo:** Script SQL (MySQL)

**Cómo ejecutarlo:**

1. Abrir MySQL Workbench
2. Seleccionar una conexión activa
3. Ejecutar el script completo o por bloques

---

## Ejercicio 2 – Base de datos Sakila

### Objetivo

Demostrar dominio del lenguaje SQL mediante consultas sobre la base de datos **Sakila**, una base de datos relacional de ejemplo que simula un sistema de alquiler de películas.

---

### Archivo: `bda-modulo-2-evaluacion-ejercicio2-bbdd-sakila.sql`

**Tipo:** Script SQL (MySQL)

**Cómo ejecutarlo:**

1. Importar previamente la base de datos Sakila en MySQL
2. Abrir el archivo en MySQL Workbench
3. Ejecutar las consultas individualmente o por bloques

---
